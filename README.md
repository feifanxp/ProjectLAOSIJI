# ProjectLAOSIJI

这是一个将学习与做事过程“RPG 化”的任务引导系统。  
用户输入问题后，系统会把目标拆成类似游戏任务的结构：主线任务、支线任务、阶段提示与关键卡点处理，帮助用户一步步推进，直到完成最终目标（击败 BOSS）。

## 项目目的

- 用游戏化机制降低复杂问题的执行门槛
- 把“我不知道从哪开始”转化为可执行的任务路径
- 通过动态提示让用户持续推进，而不是一次性给出大段教程
- 在用户卡住时自动生成新任务，形成可恢复、可继续的学习流程

## 核心机制

### 1) 主线任务（Main Quest）

- 代表最终目标达成路径
- 按阶段拆分为可执行步骤
- 每个主线节点都有“做法提示”和完成状态

### 2) 支线任务（Side Quest）

- 用于补充主线所需的前置知识、工具准备或风险处理
- 不打断主线目标，但可显著降低主线难度
- 支持按需展开，避免信息过载

### 3) BOSS 目标

- 主线最终节点即 BOSS
- 当关键主线阶段都完成后，用户进入 BOSS 收尾阶段
- BOSS 通过“目标验收标准”来判断是否真正完成

### 4) 弹性教程（Elastic Tutorial）

这是本项目的关键概念：

- 对用户“已了解”的内容：简述即可，不重复讲解
- 对用户“卡住”的内容：自动生成新的小任务与提示
- 教程强度随用户状态动态调整，做到“懂的少讲，不懂的细讲”
- 目标不是讲完知识，而是帮助用户把任务做完

## 用户体验流程（RPG 风格）

1. 输入问题（领取任务）
2. 生成主线与支线（打开任务地图）
3. 执行当前节点（获取步骤提示）
4. 遇到卡点（触发弹性教程并生成新任务）
5. 完成阶段节点（推进主线）
6. 挑战并完成 BOSS（达成最终目标）

## 当前能力

- 输入问题后按需拆解任务树
- 支持节点级继续拆解（递归展开）
- 支持任务完成状态切换
- 支持豆包与 DeepSeek 模型切换

## 本地运行

1. 启动后端
```
cd server
cp .env.example .env
# 填入你的 API Key 与模型配置
npm install
npm run dev
```

2. 启动前端
```
cd frontend
npm install
npm run dev
```

前端地址：`http://localhost:5173/`
